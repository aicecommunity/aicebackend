User Authentication API Documentation

Base URL: http://127.0.0.1:8000/user

1. Signup

Create a new user account.

Endpoint: POST /signup/

Request Body (JSON):

{
  "first_name": "Olawale",
  "middle_name": "Ariike",
  "last_name": "Akin",
  "country_of_residence": "Nigeria",
  "whatsapp_number": "+2348012344312",
  "email": "Olawale@example.com",
  "password": "Olawale@123",
  "confirm_password": "Olawale@123"
}

Response (201 Created):

{
  "message": "User created successfully",
  "user_id": 1
}

2. Login

Authenticate an existing user and get tokens.

Endpoint: POST /login/

Request Body (JSON):

{
  "email": "Olawale@example.com",
  "password": "Olawale@123"
}

Response (200 OK):

{
  "access": "",
  "refresh": ""
}

3. Refresh Token

Refresh the access token using a valid refresh token.

Endpoint: POST /token/refresh

Request Body (JSON):

{
  "refresh": ""
}

Response (200 OK):

{
  "access": ""
}

4. Logout

Invalidate the refresh token and log out the user. Use the access token as the Bearer token in the request header and send the refresh token in the body.

Endpoint: POST /logout/

Headers:

Authorization: Bearer

Request Body (JSON):

{
  "refresh": ""
}

Response (200 OK):

{
  "message": "User logged out successfully"
}

5. Profile

Retrieve, update, or delete the authenticated user's profile.

Endpoints:

  • GET /profile/ - Retrieve user profile
  • PUT /profile/ or PATCH /profile/ - Update profile details
  • DELETE /profile/ - Delete user account

Headers:

Authorization: Bearer

Sample Update Request Body:

{
  "first_name": "Olawale",
  "country_of_residence": "Ghana",
  "whatsapp_number": "+2338012344312"
}

6. Change Password

Change the password of the authenticated user.

Endpoint: PUT /change-password/

Headers:

Authorization: Bearer

Request Body (JSON):

{
  "old_password": "Olawale@123",
  "new_password": "Olagunju@456",
  "confirm_password": "Olagunju@456"
}

Response (200 OK):

{
  "details": "Password updated successfully."
}

Important Notes

  • All endpoints require JSON-formatted requests.
  • Include the access token in the Authorization: Bearer <token> header for authenticated endpoints.
  • Use HTTPS in production for security.
